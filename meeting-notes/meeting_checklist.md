# TDWG Standards Mapping Task Group Meeting Checklist

A checklist for hosting meetings of the TDWG Standards Mapping Task Group

## Before the Meeting

### Find a suitable date

(if the meeting date was not already decided in the previous meeting)

1. Use Doodle https://doodle.com or an [alternative](https://alternativeto.net/software/doodle/)

2. Check for suitable time slots using https://www.timeanddate.com/worldclock/meeting.html

3. Pick dates that are at least 1 month in the future to ensure sufficient availability from the other participants
   
   - Meetings are usually 1 - 1.5 hours, so select the time slots accordingly

4. Announce the poll in the slack channel and give participants enough time to respond (1-2 weeks)
   
   - Use the @Channel tag to directly inform all the people in the channel. 

5. Send a reminder in the Slack channel one or two days before the deadline.

6. Select a suitable date and close the Doodle

### Setting the Agenda and Announcing the date

1. Ask Stan Blum to create a Zoom room for the meeting.
2. Create a draft agenda of the meeting
   - check the previous minutes for open To-dos and postponed agenda points
3. Add the details in the running minutes file in Google Docs
4. Announce the Meeting with the main agenda points and Zoom link on Slack
   - Invite people to contribute topics to the agenda, if they have any
5. Create a ticket on Github* for the meeting
   - https://github.com/tdwg/standards-mappings/issues , use the meetings issue template
   - Don't post the Zoom link directly, instead point to the Slack channel
6. Optional: if you want to, you can prepare slides for the agenda points

`*` if you don't have a Github account, ask David or one of the other group members do create them for you (or create one and ask for access to the repository)

## On the day of the conference

1. Send a reminder via Slack

## During the Conference

1. Start the Zoom Sessions 10 minutes early and check your setup
2. Start the recording
3. Ask participants to add their names to the participants list and help you keep notes
4. Go through the agenda, moderate the discussions and keep the time in mind
5. Keep track of actionable To-dos and who is responsible for them
6. End the recording

## After the conference

1. Complete the minutes
2. Export the minutes to MarkDown (use [Google Doc to Markdown Converter](https://gdoc2md.com/) and [the template file](https://github.com/tdwg/standards-mappings/blob/main/meeting-notes/template_meeting_notes.md)) and publish them on Github
   - add any files to the repo as well (slides, images, etc.), stay with the convention that file names start with the date as yyyy-mm-dd 
3. Post the link to minutes with a short summary of the outcome in the Slack channel
4. Close the Github Ticket for the meeting and link to the minutes file
